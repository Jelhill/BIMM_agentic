Movie Watchlist Manager — Feature Spec

A React + TypeScript single-page application for browsing and managing a personal movie watchlist. Built with Vite, MUI, Apollo Client, and MSW for mock GraphQL.

## Data Model

Each movie has: id, title, genre, year, rating (number 1-10), watched (boolean), and poster (URL string).

## Features

### 1. Movies List (GraphQL + MSW Mock API)
- Use Apollo Client to fetch movies via a `GET_MOVIES` GraphQL query.
- Mock the GraphQL API using MSW with a handler that returns a static list of at least 5 sample movies.
- Display the list of movies as MUI Cards in a responsive grid layout.

### 2. Responsive Images
- Each movie card displays a poster image that adapts to viewport width:
  - Mobile (≤640px): images render at full width of card
  - Tablet (641–1023px): images render at medium size within a 2-column grid
  - Desktop (≥1024px): images render at fixed size within a 3- or 4-column grid
- Use MUI's `useMediaQuery` or responsive Grid props for breakpoints.

### 3. MUI Movie Cards
- Each card shows: title, genre, year, rating, watched status, and poster.
- Use MUI `Card`, `CardMedia`, `CardContent`, and `Typography` components.
- Show a MUI `Chip` indicating whether the movie is watched or unwatched.
- Show the rating as a numeric badge or MUI `Rating` component.

### 4. Add Movie Form (GraphQL Mutation)
- A form (dialog or inline) to add a new movie with fields: title, genre, year, rating, watched, poster URL.
- Submits via an `ADD_MOVIE` GraphQL mutation.
- Mock the mutation in MSW to append the new movie to the in-memory list.
- On success, the movies list updates to include the new movie.

### 5. Search and Sort
- A search input that filters the displayed movies by title (case-insensitive substring match).
- Sort controls to sort by year (ascending/descending) or rating (high to low).
- A toggle to filter by watched vs unwatched vs all.
- Filtering and sorting operate on the client side over the fetched data.

### 6. useMovies() Custom Hook
- Encapsulate all GraphQL logic (query, mutation, cache update) in a `useMovies()` custom hook.
- The hook returns: `{ movies, loading, error, addMovie, refetch }`.
- Components consume this hook instead of calling Apollo directly.

### 7. Unit Tests
- Test the movies list rendering with mocked Apollo provider.
- Test the Add Movie form submission and list update.
- Test search filtering and sort behavior.
- Test the useMovies() hook using renderHook with a mocked provider.
- Use Vitest + React Testing Library.

## Tech Stack
- React 19, TypeScript, Vite
- MUI v6 (Material UI)
- Apollo Client + GraphQL
- MSW v2 for API mocking
- Vitest + React Testing Library for tests
